# FND-002 — [accessibility/contrast] Disabled input border is indistinguishable from resting (and inverted in dark)

**Severity:** medium
**Project:** R&T Deposit Solutions / R&T Connect
**Location:** AddUser screen → disabled text inputs and textareas (`.form-control[data-input][disabled]`).

## Observed (as designed)
Disabled inputs take `border-color: var(--color-neutral-3)` (`#d9d9d9`). With FND-001 applied, the resting border is now also `#d9d9d9`, so in light theme **the disabled and resting borders are the same color**. Disabled is distinguished only by its `--color-background-secondary` fill and the `not-allowed` cursor.

Dark theme is worse. `--color-neutral-3` is never overridden under `[data-theme="dark"]`, so it stays `#d9d9d9`, while the resting border becomes `#444444`:

| State | Dark border | On `#1e1e1e` |
|---|---|---|
| resting (enabled) | `#444444` | 1.71:1 |
| disabled | `#d9d9d9` | 11.81:1 |

The disabled control therefore draws **more** visual emphasis than the editable one — the opposite of the intended hierarchy.

## Rule
No success criterion is violated: SC 1.4.11 explicitly exempts inactive/disabled components. This is a design-intent and state-legibility issue, raised because resolving it means choosing a color the design has not specified.

Figma Frame 439 contains no disabled input, so there is no as-designed value to build against.

## Recommendation (for design / brand owner)
1. Specify a disabled border in the frame. A value one step lighter than resting in light theme (and one step *darker* in dark theme) preserves the hierarchy.
2. Independently of that, decide whether `--color-neutral-3` should gain a `[data-theme="dark"]` override, or whether disabled should stop consuming a raw neutral and move to the existing `--color-border-disabled` semantic token (already defined as `--rnt-neutral-600` in dark, unset in light).

The second point is the real defect — a raw Tier-1-facing neutral is doing a semantic job, which is why it fails to flip with the theme.

## Implementation note
Left exactly as-is (`--color-neutral-3`). Not auto-corrected: any fix picks a color the design does not specify, and repointing to `--color-border-disabled` would change light-theme rendering. Surfaced here for a design decision.
